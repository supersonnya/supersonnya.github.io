Shell编程基础

Sep 01, 2022

Reading time: 28 minutes.

函数

基本介绍

1、什么是函数

  • 对于一个单独的 Shell 脚本来说，在执行的时候，会为其创建一个新的Shell 进程，来解释并执行脚本中的代码，当脚本执行完成，该 Shell 进程就会自动结束；而对于一个函数来说，在调用的时候，系统并不会为其单独创建一个 Shell 进程，而是在调用者的 Shell 进程中直接解释并执行函数中的代码。

2、函数的定义

  • Shell规定，函数必须在调用前定义。
[function] function_name ()
{
   statement1
   statement2
   ...
   statement n
}

3、函数的调用

  • 通过函数名来调用该函数：function_name param1 param2 …。
  • 当用户定义一个函数后，该函数就成为一个合法的Shell命令，可在后继的脚本中使用。定义函数时需要使用圆括号，调用函数时无需圆括号。

4、函数链接

  • 函数链接指在某个Shell函数中调用另外一个函数的过程，Shell允许用户函数的嵌套调用。
  • 函数链接不仅仅局限于在某个函数中调用另外一个函数，还可进行多层嵌套调用，或在某个函数中调用多个函数。

5、函数的返回值

  • 用户可使用return语句返回某个数值，在Shell中，return只能返回0～255之间的整数值。
  • 在函数中，用户将需要返回的数据写入到标准输出，通常使用echo语句完成，然后在调用程序中将函数的执行结果赋给一个变量。通过echo可以将各种数据作为返回值返回给函数调用者，而不仅仅局限于整数。

6、函数和别名

  • 与函数相比，别名的功能比较简单，不能为一组命令指定别名，别名中不能通过系统变量操作参数列表。

函数参数

1、含有参数的函数的调用方法

  • function_name arg1 arg2 ...。

2、获取函数参数的个数

  • 在函数内部，通过位置变量来接收参数的值。
  • 通过系统变量$#来获取脚本的参数的个数。
  • 参数以空格隔开，如果某个参数中含有空格，应该使用引号将其引用起来。

3、通过位置变量接收参数值

  • 使用位置变量来获取参数值，$0表示脚本名称，$1表示第1个参数，$2表示第2个参数等。通过系统变量$@和$*获取所有参数的值。

4、移动位置参数

  • 在函数中，用户可以使用shift命令来使得脚本的所有的位置参数向左移动一个位置，使用户可通过9以内的位置变量来获取超过9个的参数。
  • shift命令会影响到系统变量$#的值。

5、通过getopts接收函数参数

  • getopts是bash内置的一个命令，该命令可以获取函数的选项以及参数值，或者是脚本的命令行选项以及参数值，基本语法：getopts optstring [args]。参数optstring包含一个可以被getopts命令识别的选项名称列表。如果选项名称的后面带冒号，表示可为该选项提供参数值，参数值将被保存到一个名称为$OPTARG的内置系统变量中。getopts命令会依次遍历每个选项，选项名称将被保存到args变量中。

6、间接参数传递

  • 间接参数传递指通过间接变量引用来实现函数参数的传递。如果某个变量的值又是另外一个变量的变量名，则该变量称为间接变量。
  • 变量的间接引用语法：${!var_name}。

7、通过全局变量传递数据

  • 参数的作用是在主程序和函数之间传递数据。除了使用参数传递数据外，还可通过全局变量来传递。

8、传递数组参数

  • 严格地讲，Shell不支持将数组作为参数传递给函数，但可通过一些变通的方法实现数组参数的传递。
#! /bin/bash 
 
func() 
{ 
   echo "number of elements is $#." 
   while [ $# -gt 0 ] 
   do 
      echo "$1" 
      shift 
    done 
} 
a=(a b "c d" e)
## 如果将${a[@]}外面的双引号去掉，第3个元素会误认为是2个元素
func "${a[@]}"

函数库文件

1、函数库文件的定义

  • 脚本与库文件之间的唯一区别在于函数库文件通常只包括函数，脚本中可以既包括函数和变量的定义，又包括可执行的代码。函数库文件与普通的脚本的结构完全相同。
  • 通常情况下应该为函数库文件提供有意义的名称，为了便于管理应将所有的库文件单独放到一个目录中。
  • 函数库文件是由主程序载入并执行的，用户无需拥有库文件的执行权限，只要拥有读取权限即可。

2、函数库文件的调用

  • 当库文件定义好后就可以在程序中载入库文件，并且调用其中的函数，语法为：. filename。
  • 库文件可使用相对路径，也可使用绝对路径。圆点命令和库文件名之间有一个空格。
  • 使用函数库文件时，必须在调用函数前将库文件载入。

递归函数

  • Linux的Shell支持函数的递归调用，函数可以直接或者间接地调用自身。在函数的递归调用中，函数既是调用者，又是被调用者。
#! /bin/bash 

fact() 
{ 
   local n="$1" 
   if [ "$n" -eq 0 ] 
   then 
      result=1 
   else  
      let "m=n-1" 
      fact "$m" 
      let "result=$n * $?" 
   fi 
   return $result 
}  
fact "$1" 
echo "Factorial of $1 is $?"

数组

定义数组

  • 数组是指将具有相同类型的若干变量按照一定的顺序组织起来的一种数据类型。Shell语言对于数组的支持非常强大，可通过多种方式来创建一个数组。

1、通过指定元素值来定义数组

  • 数组的下标从0开始，基本语法：array[key]=value，可定义名称为array的新数组变量。

2、通过declare语句定义数组

  • 基本语法：declare -a array。
  • 实际上这样的声明不是非常必要的。在Shell中，所有的变量都不必显式定义就可以用做数组。在某种意义上讲，所有的变量都是数组，赋值给没有下标的变量与赋值给下标为0的元素的效果是相同的。

3、通过元素值集合定义数组

  • 基本语法：array=( v0 v1 v2 ... vn)。

4、通过键值对定义数组

  • 基本语法：array=([0]=value0 [1]=vlaue1 [2]=value2 ... [n]=valuen)。
  • Shell中的数组的索引并不仅仅是数字，还可以是字符串，称为关联数组。在使用关联数组时，需要首先使用 declare语句来声明数组，然后再通过键值对的形式为数组元素提供值：declare -A array。
  • 关联数组在bash 4.0及以上版本获得支持。

5、数组和普通变量

  • 在Shell中，所有的普通变量都可以当做数组变量来使用。对普通变量操作与对相同名称的下标为0的元素的操作是等效的。

数组的赋值

1、按索引为元素赋值

  • 基本语法：array[n]=valuen。

2、通过集合为数组赋值

  • 通过集合为数组赋值与通过集合定义数组的语法完全相同。当为某个数组提供一组值时，会从第1个元素开始，依次将这些值赋给每个元素。当新的值的个数超过原来的数组长度时，会在数组末尾追加新的元素；当新的值的个数少于原来的数组长度时，会将新的值从第1个元素开始赋值，然后删除超出的元素。

3、在数组末尾追加新元素

  • 在通过索引为数组元素赋值时，如果指定的索引不存在，会自动添加一个新的元素，并将指定的值赋给该元素。
  • 无论以何种方式为关联数组赋值，都需要显示指定元素的下标。

4、通过循环为数组元素赋值

  • 最常用的数组赋值方法是通过一个循环语句逐个为每个元素提供值，可以是任意一种循环语句。
#! /bin/bash 
  
for i in {1..10} 
do 
   array[$i]=$i 
done  
echo "${array[@]}"

访问数组

1、访问第1个数组元素

  • 直接使用数组名来访问数组时，得到的是下标为0的元素的值。

2、通过下标访问数组元素

  • 基本语法：array[n]。

3、计算数组的长度

  • 基本语法：${#array[@]}或${#array[*]}。
  • 获取具体元素的长度：${#array[n]}。

4、通过循环遍历数组元素

array=(Mon Tue Wed Thu Fri Sat Sun) 
## 获取数组长度  
len="${#array[@]}" 
for ((i=0;i<$len;i++)) 
do 
   echo "${array[$i]}" 
done

5、引用所有的数组元素

  • 指定下标为@或者*，表示引用当前数组中的所有的元素：${array[@|*]}。

6、以切片方式获取部分数组元素

  • 基本语法：${array [@|*]:start:length}，得到的切片是一个字符串，并非数组。
  • 基本语法：(${array [@|*]:start:length})，得到的结果是一个数组。
  • 对数组元素进行切片：${array [n]:start:length}，得到一个子字符串。
  • 长度值超过数组或者数组元素的长度，都会截取到数组或者数组元素的末尾为止。

7、数组元素的替换

  • 基本语法：${array[@|*]/pattern/replacement}，部分内容用其他的字符串来代替，不影响原来的数组的值。
  • 为得到一个新的数组变量，需要使用圆括号操作符，否则得到一个字符串变量：a=(${a[@]/3/100})。

删除数组

1、删除指定数组元素

  • 基本语法：unset array[n]。

2、删除整个数组

  • 基本语法：unset array。

数组的其他操作

1、复制数组

  • 实现语法：newarray=("${array[@]}")。

2、连接数组

  • 实现语法：("$array1[@]}" "${array2[@]}")。

3、加载文件内容到数组

# 加载文件内容  
content=(`cat "demo.txt"`)  
for s in "${content[@]}" 
do 
   echo "$s" 
done

正则表达式

什么是正则表达式

  • 来描述某些字符串匹配规则的工具。

1、为什么使用正则表达式

  • 查找符合某些比较复杂规则的字符串时如果单纯依靠程序设计语言本身，往往需要复杂的代码来实现。使用正则表达式，则会以非常简短的代码完成。

2、如何学习正则表达式

  1. 重点在于理解元字符。
  2. 掌握好正则表达式的语法。
  3. 开拓思路，寻找最佳的表达方法。

正则表达式基础

1、正则表达式的原理

  • 正则表达式是对文本进行过滤的工具，它定义了一系列的元字符，通过元字符配合其他的字符来表达出一种规则，只有符合该规则的文本才能保留下来，不符合该规则的文本则被过滤掉。
  • 元字符是指用来描述字符的字符。元字符的作用在于对字符表达式的内容、转换以及各种操作信息进行描述。

2、基本正则表达式

  • 基本正则表达式（Basic Regular Expression，BRE），又称为标准正则表达式，是最早制订的正则表达式规范，仅支持最基本的元字符集。
^ ## 行首定位符,匹配行首的字符,定位符用来匹配某个具体的位置
grep "^app"
$ ## 行尾定位符
grep "conf$"
## 完全匹配cat
^cat$
## 匹配所有的空行,行中没有任何字符，包括空白字符
^$
## 匹配所有行，包括^$表示的空行
^ | $
. ## 匹配任意单个字符，包括空格，不包括换行符,可连续使用圆点符号来匹配多个字符
## 限定符本身不代表任何字符，指定其前面的一个字符必须重复出现多少次才能满足匹配
* ## 匹配其前导字符的任意次数，包括0次
[] ## 字符集匹配,指定一个字符集合
## 元字符*或者.等位于字符集匹配符后，变成普通字符，只有字面意义，没有特殊意义
[abc] | [a-f] | [0-9]
[^] ## 字符集不匹配
[^abc]

3、扩展正则表达式

  • 扩展正则表达式（Extended Regular Expression，ERE）支持比基本正则表达式更多的元字符，但扩展正则表达式对有些基本正则表达式所支持的元字符并不支持。
  • 扩展正则表达式取消了子表达式()和次数匹配{m,n}语法符号的转义符引用语法，在使用这两种语法符号时，不再需要添加转义符。
+ ## 限定符,限定前面的字符至少出现1次
? ## 限定符,限定前面的字符最多只出现1次
| ## 表示多个正则表达式之间 或 的关系
() ## 表示一组可选值的集合,|和()经常在一起使用，表示一组可选值

4、Perl正则表达式

  • Shell中的grep和egrep命令都支持Perl正则表达式。Perl正则表达式的元字符与扩展正则表达式的元字符大致相同，扩展正则表达式中的元字符在Perl正则表达式中都得到了支持。Perl正则表达式还增加了一些元字符。
\d ## 匹配0～9中的任意一个数字字符，等价于[0-9]
\D ## 匹配一个非数字字符,等价于表达式[^0-9]
\s ## 匹配任何空白字符，包括空格、制表符以及换页符,等价于[\f\n\r\t\v]
\S ## 匹配任何非空白字符，等价于[^\f\n\r\t\v]

5、正则表达式字符集

  • POSIX字符类也需要放在方括号：[[:alnum:]]。
## 常用的POSIX字符类 
[:alnum:] ## 匹配任意一个字母或者数字，等价于A-Za-z0-9 
[:alpha:] ## 匹配任意一个字母，等价于A-Za-z 
[:digit:] ## 匹配任意一个数字，等价于0-9 
[:lower:] ## 匹配任意一个小写字母，等价于a-z 
[:upper:] ## 匹配任意一个大写字母，等价于A-Z 
[:space:] ## 匹配任意一个空白字符，包括空格、制表符、换行符，以及分页符等  
[:blank:] ## 匹配空格和制表符  
[:graph:] ## 匹配任意一个看得见的可打印字符，不包括空白字符  
## 匹配任何一个可以打印的字符，包括空白字符
## 不包括控制字符、字符串结束符 '\0' 、 EOF文件结束符（ -1 ） 
[:print:]
[:cntrl:] ## 匹配任何一个控制字符，即ASCII字符集中前32个字符 
[:punct:] ## 匹配任何一个标点符号，例如[]、{}或者,等  
[:xdigit:] ## 匹配十六进制数字，即0-9、a-f以及A-F

正则表达式应用

1、匹配单个字符

  • 单个一般字符：除了正则表达式中已经定义的元字符之外的所有字符，直接将该字符作为表达式或者是表达式的一部分。
  • 转义后的元字符：想要匹配元字符本身，需要在这些字符的前面加上转义字符\，关闭元字符的特殊意义，只保留其字面意义。\也是一个元字符，匹配反斜线，也需要在其前面加上转义字符。
  • .表达式和[]表达式。
  • 当元字符位于方括号中时，除了极少的几个元字符，例如-或^外，其他元字符都将失去其特殊意义，只保留字面意义。

2、匹配多个字符

  • 将多个字符按照指定的顺序拼接起来：matter。
  • 将普通字符和其他字符集混用：o[ru]。

3、匹配字符串的开头或者结尾

  • 定位符仅仅影响到其后置字符或者前置字符，而不是一个字符串。

4、运算符优先级

## 正则表达式运算符优先级 
\  ## 转义符  
[]  ## 方括号表达式  
()  ## 分组  
* 、 + 、 ? 、 {m} 、 {m,} 、 {m, n}  ## 限定符  
普通字符   ## 按照从左到右的顺序  
^ 、 $  ## 定位符  
|  ## 或运算

5、子表达式

  • 子表达式，是指由多个普通字符或者元字符组成的一个小的正则表达式。子表达式本身也是一个完整的表达式，在使用时作为一个大的正则表达式的一部分来使用的，而不是单独使用。在正则表达式中，子表达式作为一个整体来看待。子表达式使用()括起来。
  • 使用圆括号将表达式分组，对执行结果没有影响，但会影响表达式的解释效率。

6、通配符

  • Shell本身不支持正则表达式，使用正则表达式的是一些Shell命令和工具，Shell使用了正则表达式中的某些元字符作为其通配符，常用的有*、?、[]、{}、^等，这些字符在Shell中的意义与在正则表达式中的意义有些区别。例如ex*、[de]*、ex8-[1-9].sh。

grep命令

1、grep命令的基本语法

## 从一个或者多个文件中搜索满足指定模式的文本行，并且打印出来
grep [options] pattern [file…]
## 常用的grep 命令options    ## 
-c ## 只打印匹配的文本行的行数，不显示匹配的内容  
-i ## 匹配时忽略字母的大小写  
-h ## 当搜索多个文件时，不显示匹配文件名前缀  
-l ## 只列出含有匹配的文本行的文件的文件名，而不显示具体的匹配的内容  
-n ## 列出所有的匹配的文本行，并显示行号  
-s ## 不显示关于不存在或者无法读取文件的错误信息  
-v ## 只显示不匹配的文本行  
-w ## 匹配整个单词  
-x ## 匹配整个文本行  
-r ## 递归搜索，不仅搜索当前目录，还要搜索其各级子目录  
-q ## 禁止输出任何匹配结果，以退出状态码的形式表示搜索是否成功，0表示找到了匹配的文本行  
-b ## 打印匹配的文本行到文件头的偏移量，以字节为单位  
-E ## 支持扩展正则表达式  
-P ## 支持 Perl 正则表达式  
-F ## 不支持正则表达式，将模式按字面意义匹配

## pattern表示要匹配的模式,可以是字符串、变量、正则表达式
## 如果模式中含有空格，需要将模式使用双引号引起来

## file表示一系列的文件名，模式后面的所有的字符串参数都被看做文件名

2、grep命令族简介

  • grep命令族已经包括grep、egrep、fgrep等3个命令。egrep是grep的扩展，使用扩展正则表达式作为默认的正则表达式引擎，egrep支持更多的元字符。fgrep是fixed grep或fast grep的缩写。在fgrep中，所有的正则表达式中的元字符都将作为一般字符，仅拥有字面意义，不再拥有特殊意义。
