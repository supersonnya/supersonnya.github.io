Database Redis

Dec 09, 2022

Reading time: 10 minutes.

NoSQL数据库

Web发展历史

  • Web1.0：以门户网站为代表，主要以用户阅读、浏览为主，没有太大性能问题。
  • Web2.0：用户参与，网站和数据库压力越来越大。
  • 数据库性能瓶颈：单机数据库的性能有限，需要使用Redis作为缓存数据块，解决高并发问题缓解数据库的压力。

NoSQL数据库

  • 非关系型数据库或Not Only SQL，不保证ACID特性。
  • 特点：非关系型；没有模式，可存储结构化、非结构化或半结构化数据；速度快；支持海量数据存储。
  • 应用场景：高并发读写、海量数据读写、高可扩展、速度快。
  • 不适用场景：事务支持、结构化查询。
  • NoSQL数据库：memcache、Redis、mongoDB。
  • NewSQL：基于NoSQL又可支持SQL操作，例如HBase。

Redis介绍

基本介绍

  • 开源的、使用ANSI、C语言编写的key-value数据库。
  • 与Memcache类似，数据缓存在计算机内存中。

应用场景

  • 取出最新N个数据。
  • 排行榜应用，取出TOP N操作。
  • 需要精准设定过期时间的应用，例如短信验证码。
  • 计数器应用。
  • 获取某段时间所有数据去重值。
  • 实时系统，反垃圾系统，用于存储一些规则。
  • 缓存，数据之间存放在内存中。

特点

  • 高效性：高速读写。
  • 原子性：所有操作都是原子性的。
  • 支持多种数据结构：字符串、列表、集合、哈希、有序集合。
  • 稳定性：持久化、主从复制集群。
  • 其他特性：过期时间、事务、消息订阅。

Redis安装

## 获取安装包redis.tar.gz
## 解压安装包
tar -zxvf redis.tar.gz
## 安装C程序运行环境
yum install gcc-c++ -y
## 安装新版本tcl，控制Linux中命令的执行流程
yum install tcl -y
## 编译redis
make
make test && make install
## 修改redis配置文件
mkdir -p ./redis-x.x.x/log
mkdir -p ./redis-x.x.x/data
vi redis.conf
bind ip_addr
daemonize yes
logfile "./redis-x.x.x/log/redis.log"
dir ./redis-x.x.x/data
## 启动redis
bin/redis-server redis.conf
## 关闭redis
bin/redis-cli -h host_name shutdown
## 连接redis
bin/redis-cli -h host_name
## Windows客户端Redis Desktop Manager
  • 生产环境关闭Redis时，不要使用kill命令，否则可能导致数据丢失。

BitMaps操作

  • 针对位操作，更加节省空间，不是一种数据结构，操作是基于String结构的。
  • 与使用字符串的方法不相同，类似于一个以位为单位的数组，数组的每个元素只能存储0或1，数组下标称为平移量。
  • 应用：可将每个独立用户是否访问过网站存放在BitMaps中，访问的用户记为1，未访问的记为0，偏移量作为用户ID。
  • 应用以指定数字开头时偏移量过大会导致空间浪费，每次操作时减去这个指定数字。
  • 第一次初始化时，如果偏移量比较大，初始化过程会比较慢，可能导致Redis阻塞。
SETBIT key offset value # value只能取0或1，offset从0开始
GETBIT key offset
BITCOUNT key [start end]
BITOP operation destkey key [key ...] # AND交集、OR并集、NOT、XOR

HyperLogLog结构操作

  • 只用于大数据量的统计，存在一定误差，提供不精确的去重计数，例如页面访问量或用户访问量统计。
  • Redis实现HyperLogLog算法，在2.8.9版本添加HyperLogLog结构。
  • Redis HyperLogLog是用于技术统计的算法，在输入元素的数量或体积非常大时，计算基数所需的空间时很小且固定的，每个键使用12kb内存计算接近2^64个不同元素的基数。只会根据输入元素计算基数，不存储输入元素本身。
PFADD key element [element ...]
PFCOUNT key [key ...]
PFMERGE destkey sourcekey [sourcekey ...]
