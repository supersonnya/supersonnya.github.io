Linux常用命令

Aug 04, 2022

Reading time: 12 minutes.

命令

  • 要在命令后加上&，指导程序在后台运行

> | > >

  • >: 把程序的输出写入目标文件，若文件不存在则新建，覆盖原来内容
  • >>: 把程序的输出追加到目标文件

< | < <

  • <: 输入重定向，让程序从一个文件中获取输入
  • <<: 立即文档，指定一个代表输入结束的分隔符，遇到这个单词的时候，即认为输入结束，并把键盘输入一起传递给命令
  • 输入重定向和输出重定向可结合在一起使用

|

  • 管道，将一条命令的输出连接到另一条命令输入

cat | more | less

  • cat: 查看文件内容，后跟文件名作为参数
    • -n: 在每一行前显示行号
  • more: 一页一页地显示文件内容
    • 空格键向下翻动一页，Enter键向下滚动一行，Q键退出
  • less: 更好的文本阅读工具
    • -M: 阅读文件时在底部显示更多的文件信息
    • 空格键向下翻一页；B键向上翻一页；/跟上查找的内容搜索某个字符串，继续查找相同的内容，再次输入/，并按下回车键；Q键退出

cd

  • 改变目录

chgrp

  • 设置文件的属组
  • -R: 改变一个目录及其下所有文件和子目录的所有权

chmod

  • 改变文件的权限，只有文件的属主和root用户才有权修改文件的权限
  • 用户组包括文件属主u、文件属组g、其他人o和所有人a，权限包括读取r、写入w和执行x
  • 设置权限规则：用户组+/-权限、用户组=权限、用户组1=用户组2
  • 可使用八进制语法设置文件权限

chown [OPTION]… [OWNER][:[GROUP]] FILE…

  • 改变文件的所有权，参数OWNER或:GROUP可省略
  • -R: 改变一个目录及其下所有文件和子目录的所有权

cp

  • -i: 目标目录有同名文件时提示是否覆盖
  • -b: 在目标目录的同名文件的文件名后加~
  • -r: 将子目录连同其中的文件一起复制到另一个子目录下

df

  • 查看磁盘使用情况
  • -t: 用于显示特定的文件系统

dir | vdir

  • dir: 列出目录内容
  • vdir: 相当于为ls命令加上-l选项

fdisk

  • 建立分区表
  • m显示所有可用的命令及其简要介绍

find [OPTION] [path…] [expression]

  • 在指定范围内查找文件
  • -name: 指定文件名
  • -type: 指定文件类型
  • -print: 表示将结果输出到标准输出
  • -atime +[-]n: 最近n天[没有]使用过的文件
  • -mtime +[-]n: 最近n天[没有]修改过的文件

fsck

  • 检查和修复文件系统，使用fsck检查并修复文件系统是存在风险的，特别是当磁盘错误非常严重的时候
  • fsck 分区名: 指定需要检查的文件系统
  • fsck -p: 会读取fstab文件来确定检查哪些文件系统
  • -f: 强制检查新的文件系统

grep [OPTIONS] PATTERN [FILE…]

  • 查找文件内容，返回所在行
  • 搜索内容包含空格时必须加单引号

groupadd

head | tail

  • 显示文件的开头和结尾
  • -n: 指定显示的行数

history

  • 查看历史操作，仅在BASH中适用，只能列出当前用户的操作记录
  • 管理员需要查看其他用户的操作记录，可读取该用户主目录下的.bash_history
  • history n: 列出最近使用的n条命令

id

  • 查看当前用户的UID、GID及其所属的组
  • id username: 查看指定用户的信息

ifconfig

  • 不带参数显示当前系统上所有网络接口的配置
  • 启动或禁用一个网络接口，设置IP地址、子网掩码以及其他网络选项
  • ifconfig eth0 192.168.1.14 netmask 255.255.255.0 up

kill [-signal] pid

  • kill -l: 显示所有信号及其编号
  • 向进程发送信号，信号用于控制进程的行为，默认情况下，向进程发送TERM信号
  • sudo kill -KILL pid: KILL可在内核级别杀死一个进程，在绝大多数情况下，下可以确保结束进程号为pid的进程。

locate

  • 更快速地定位文件，自动建立整个文件名数据库

ln TARGET LINK_NAME

  • 创建硬链接，硬链接是直接引用，符号链接通过名称进行引用
  • -s: 创建软链接，可用于文件和目录

ls [OPTION]… [FILE]…

  • 列出目录内容
  • -F: 在每个目录后加上/，在可执行文件后加*，在链接文件后加上@
  • -a: 用于显示所有文件
  • -l: 查看文件的各种属性

lsof

  • 查看占用文件的进程，不带任何参数的lsof会列出当前系统中所有打开文件的进程信息，要找出占用某个特定文件的进程，需要提供文件名作为参数。

lsusb

  • 列出当前内核已经发现的USB设备

man

  • 获取某个命令的帮助信息

mkdir

  • 一次建立一个或几个目录
  • -p: 完整创建目录结构，不存在父级目录时自动创建

mkfs

  • 建立文件系统，如果硬盘分区已经挂载到文件系统中，格式化之前必须用umount 卸载该分区
  • -t: 指定建立的文件系统类型
  • -c: 检查指定设备上损坏的块

mkswap

  • mkswap /dev/sdb2: 初始化交换分区
  • swapon /dev/sdb2: 检查并激活交换分区
  • swapon -s: 查看当前系统上已存在的交换分区

mount

  • 用户不能直接通过设备文件访问存储设备，所有的存储设备（包括硬盘、光盘等）在使用之前必须首先被挂载到一个目录下
  • mount /dev/cdrom /mnt/cdrom/: 挂载光盘
  • -t: 指明设备所使用的文件系统类型
  • -r: 以只读模式挂载设备
  • -w: 可读写模式挂载设备

mv

  • -i: 目标目录有同名文件时提示是否覆盖
  • -b: 在目标目录的同名文件的文件名后加~

netstat

  • netstat -r: 查看当前系统中的路由信息

nice

  • 查看默认谦让度
  • nice -n 2 bc: 启动程序时设置其谦让度增量，谦让度的值在–20～+19
  • renice +12 -p 8567: 在进程运行时调整其谦让度绝对值

passwd [username]

ps

  • 监视进程
  • ps aux: 显示当前系统上运行的所有进程的信息
  • ps lax: 父进程 ID（PPID）和谦让度（NI）

pwd

  • 显示当前目录

rm

  • 一次删除一个或几个文件，可用来删除目录
  • -i: 删除文件前给出提示
  • -f: 强制执行删除操作不给出提示
  • -r: 递归删除目录下所有的文件和子目录

rmdir

  • 只能删除空目录

route

  • route add default gw xx.xx.xx.xx: 增加一条默认路由
  • route add -net xx.xx.xx.xx/24 gw xx.xx.xx.xx [eth0]
  • route add -host xx.xx.xx.xx gw xx.xx.xx.xx
  • route del default: 删除默认路由

su

  • 用户间切换，不带参数的su会将用户提升至root权限，su获得的特权将一直持续到使用exit退出为止。
  • Ubuntu默认情况系统没有合法的 root 口令，不能使用su提升至root权限，必须用sudo获得root访问权。
  • su username: 切换到其他用户。
  • 尽量通过绝对路径使用su命令，通常保存在/bin 目录下。这将在一定程度上防止溜入到搜索路径下的名为 su 的程序窃取用户口令。

sudo

  • 受限的特权，管理员通过配置/etc/sudoers 指定用户可以执行的特权命令。修改sudoers文件应该使用visudo命令。
  • sudo -u username: 改变执行命令用户身份

top

  • 即时跟踪进程信息，显示的信息会占满一页，在默认情况下每10s更新一次。使用CPU最多的程序会排在最前面。使用q退出监视程序。

touch

  • 新建一个空文件
  • 更新一个文件的建立日期和时间

umount

  • 卸载前必须先退出挂载到的那个目录，统会提示设备忙并拒绝卸载
  • -r: 在无法卸载文件系统的情况下，尝试以只读方式重新载入

uname

  • 显示当前系统的版本信息
  • -a: 显示当前操作系统的所有有用信息
  • -r: 显示内核版本信息

useradd

  • -m: 为新用户建立主目录，为建立主目录可登录到Shell，不能登录图形界面
  • -g: 指定用户所属的组，不带-g默认会创建一个同名的用户组
  • -s: 指定用户登录后使用的Shell，默认将使用 sh

userdel

  • -r: 删除用户同时删除其主目录，默认不删除主目录

usermod

  • -d: 修改用户主目录
  • -e: 修改账号的有效期限，以公元月/日/年的形式表示（MM/DD/YY）
  • -g: 修改用户所属的组
  • -l: 修改用户账号名称
  • -s: 修改用户登录后所使用的Shell

whatis | apropos

  • whatis: 从某个程序的使用手册页中抽出一行简单的介绍性文字，介绍大致用途
  • apropos: 通过使用手册中反查到某个命令

whereis

  • 查找程序文件，whereis检索的子目录固定编写在程序中
  • -b: 只查找二进制可执行文件

who

  • 查看当前系统中登录的用户，以及工作的控制台

whoami
