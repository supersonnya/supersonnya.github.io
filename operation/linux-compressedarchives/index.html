压缩存档备份

Aug 08, 2022

Reading time: 5 minutes.

压缩工具

gzip

  • 压缩后只生成新压缩文件，不保留原文件
# 压缩文件
gzip <filename>
# 解压文件，支持的扩展名.gz、.Z、-gz、.z、-z、z
gunzip <filename>
gzip -d <filename>
# 查看压缩效果
gzip -l <filename>
# 测试压缩文件的完整性
gzip -tv <filename>

bzip2

  • 提供比gzip更高的压缩率，速度比gzip慢
# 压缩文件
bzip2 <filename>
# 解压文件，支持的扩展名.bz2、.bz、.tbz2、.tbz、bzip2
bunzip2 <filename>
bzip2 -d <filename>
# 测试压缩文件的完整性
bzip2 -tv <filename>

RAR for linux

# 解压文件
 rar x xxx.rar

存档工具

tar

# 文件打包，不会删除原来的文件，命令选项前的-可省略
## -c创建归档文件，-v显示命令的执行过程，-f指定归档文件的文件名
## 最后一个（或者几个）参数指定了需要打包的文件和目录
tar -cvf xxx.tar xxx/
tar -xvf xxx.tar
## -w每次将单个文件加入（或者抽出）归档文件时询问，yn回答
tar -cvwf xxx.tar xxx/
tar -xvwf xxx.tar
## -z调用gzip程序
tar -czvf xxx.tar.gz xxx/
tar -xzf xxx.tar.gz
## -j调用bzip2程序
tar -xjf xxx.tar.bz2

dd

  • 复制文件系统
  • dd if=/dev/cdrom of=CD.iso

系统备份

  • 绝大多数的增量备份工具不会注意到磁盘上已经删除的文件，当恢复数据的时候，一些已经被删除的文件会再次出现在系统上。要找到并再次清除这些垃圾会很繁琐。如果存储容量允许，可首先考虑完整备份。

dump

  • 使用dump进行增量备份时，只能在像磁带这样的字符设备（顺序访问设备）上进行。如果使用硬盘做备份，只能进行0级备份。
# 执行从/web 到/dev/nst0 的备份
## -0执行级别为0的备份，备份级别总共有0～9，0表示完整备份
## -u 指定更新/etc/dumpdates文件，这个文件中记录了历次备份
## 的时间、备份级别和实施备份的文件系统，dump在实施增量
## 备份的时候需要依据这个文件决定哪些文件应该备份
### -u要求备份的必须是一个完整的文件系统
## -f指定用于存放备份的设备
dump -0u -f /dev/nst0 /xxx
# 将备份转储到远程主机上，需要指定远程主机的主机名或IP
rdump -0u -f hostname:/dev/nst0 /xxx

restore

  • dump的配套工具，用于从备份设备中提取数据，在使用 restore 恢复数据之前，首先需要建立一个临时目录，这个目录用于存放备份设备中的目录层次，用 restore 恢复的文件也会存放在这个目录下。
mkdir /var/restore
## -i用于交互式地恢复单个文件和目录
## -f指定存放备份的设备
## 执行后进入交互式的命令行界面，用ls和cd命令在备份的文件系统中浏览，
## 需要恢复的文件，用add标记，最后使用extract提取所有做过标记的文件和目录
## 提取完成后，使用quit退出restore
restore -i -f /dev/sdb3
# 恢复整个文件系统
## 进入需要要恢复的目录
cd /xxx
## -r 恢复整个文件系统
restore -r -f /dev/sdb3
# 以交互的方式从远程主机提取备份信息
rrestore -i -f hostname:/dev/nst0

cron

  • 让备份定时自动完成，定时执行命令的软件，完成需要定期重复的工作
