MySQL基础

Aug 09, 2020

Reading time: 5 minutes.

基本操作

操作表

  • SHOW CREATE TABLE 表名：查看指定表的创建语句
  • DESC 表名：查看表结构
  • ALTER TABLE 表名 MODIFY 列名 列类型：修改列类型
  • ALTER TABLE 表名 CHANGE 原列名 新列名 列类型：修改列名
  • ALTER TABLE 原表名 RENAME TO 新表名：修改表名称

删除表

  • TRUNCATE：先删除表和数据，再按照表结构重建表

DCL

  • 一个项目创建一个用户，一个项目对应的数据库只有一个
  • 用户只对这个数据库有权限
  • 创建用户
    • CREATE USER 用户名@IP IDENTIFIED BY ‘PASSWORD’
    • CREATE USER 用户名@’%’ IDENTIFIED BY ‘PASSWORD’
  • 查看权限
    • SHOW GRANTS FOR 用户名@IP
  • 删除用户
    • DROP USER 用户名@IP

排序

  • ORDER BY：后面对多列排序，依次筛选，前面列相同时才进行后面的排序

聚合函数

  • COUNT()
  • SUM()
  • MAX()
  • MIN()
  • AVG()

分组

  • 分组时查询的列除了主信息，只能是聚合函数，否则导致逻辑错误
  • WHERE 是分组前的筛选，HAVING 是分组后的筛选

LIMIT 方言

  • SELECT * FROM tableName LIMIT 4, 3;从第5行开始，查询三行

约束

  • 约束是添加在列之上的，用来约束列

1.主键约束

  • 唯一标识
  • 非空、唯一、被引用
1.sid CHAR(6) PRIMARY KEY
2.PRIMARY KEY(sid)
3.指定sid列为主键列，即为sid列添加主键约束
4.ALTER TABLE stu ADD PRIMARY KEY(sid);
5.ALTER TABLE stu DROP PRIMARY KEY

2.主键自增长

  • 主键，整型
  • 不建议使用自然数据做主键，新建id列
  • 使用UUID做主键，不会重复
1.sid INT PRIMARY KEY AUTO_INCREMENT
2.ALTER TABLE stu CHANGE sid sid INT AUTO_INCREMENT
3.ALTER TABLE stu CHANGE sid sid INT

3.非空约束

  • NOT NULL: 不能为空

4.唯一约束

  • UNIQUE: 不能重复

5.外键

  • 外键必须引用主键
  • 外键可以重复、可以为空、可以有多个
1.CONSTRAINT fk FOREIGN KEY(deptno) REFENENCES(deptno)
2.ALTER TABLE emp ADD CONSTRAINT fk FOREIGN KEY(deptno) REFENENCES(deptno)

6.一对一关系

  • 从表的主键设为外键

7.多对多关系

  • 三张表实现
  • 两张主表不包含关系，一张关联表报错关系

多表查询

合并结果集

  • 被合并的表，列的类型和列数相同
  • UNION: 去除重复记录
  • UNION ALL: 不去除重复记录

连接查询

  • 内连接
方言：SELECT * FORM table1, table2 WHERE table1.x=table2.x
标准：SELECT * FORM table1 INNER JOIN table2 ON table1.x=table2.x
自然：SELECT * FORM table1 NATURAL JOIN table2(名称相同的列自动定位)
内连接查询出的所有记录都满足条件
  • 外连接：左外连接、右外连接、全外连接(Mysql不支持)
左：SELECT * FORM table1 LEFT OUTER JOIN table2 ON table1.x=table2.x
右：SELECT * FORM table1 RIGHT OUTER JOIN table2 ON table1.x=table2.x
全：SELECT * FORM table1 FULL OUTER JOIN table2 ON table1.x=table2.x
1.主表中记录全部显示，不满足条件时，从表使用NULL补位
2.Mysql不支持全外连接，合并左外和右外结果集实现

子查询

  • 查询中有多个SELECT
  • 出现的位置：where后作为条件；from后作为表存在
单行单列：SELECT * FORM t1 WHERE c1 (=|>|<|<=|>=|!=)(SELECT c2 FROM t2 WHERE)
多行单列：SELECT * FORM t1 WHERE c1 (=|>|<|<=|>=|!=)(IN|ALL|ANY)(SELECT c2 FROM t2 WHERE)
单行多列：SELECT * FORM t1 WHERE (c1,c2) IN (SELECT c3,c4 FROM t2 WHERE)
多行多列：SELECT * FORM t1,(SELECT ...) WHERE

备份与恢复

备份

  • mysqldump -uroot -p1234 dbname>dir
  • 不要加分号，不登录mysql，直接在cmd中运行
  • 生成的脚本不包含create database语句

恢复

  1. mysql -uroot -p1234 mydb1<filename.sql(不登录)
  2. source dir(登录后切换数据库，执行脚本)

编码

查看MySQL数据库编码

  • SHOW VARIABLES LIKE ‘char%’;
  • character_set_client=utf8: 无论客户端发送什么编码的数据，mysql都当成是utf8的数据
  • character_set_results=utf8: 把数据用什么编码发送给客户端
  • my.ini: default-character-set=gbk(client,results,connection)
